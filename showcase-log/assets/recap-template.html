// methodology

Data basis. Everything here derives from session-log/ — a structured log Claude appends to after each request (verbatim prompt, actions, decisions, model) — plus usage/, a permanent snapshot of the exact token usage Claude Code records on every assistant message. Claude Code keeps those records locally and deletes them after ~30 days; this skill copies them into the repo first, so the cost figures are harvested actuals, not estimates.

{{COST_METHODOLOGY_NOTE}}

Hours. "Est. worked" sums each logged request's start-to-end span, from real timestamps recorded at the top and bottom of the request. It counts time inside logged requests only — not idle gaps between them, and not work done outside Claude.

Requests & active days. One request = one log entry; active days = distinct calendar dates with at least one entry.

AI-written sections (workstreams, timeline, patterns, findings), where included, are Claude's interpretation of the log — not computed metrics. Their classifications and rankings reflect model judgment and are worth a sanity check.

// about this skill

showcase-log turns a project's disappearing Claude Code history into a permanent, structured log — and turns that log into cost reports and interactive recaps like this one. It's the always-on data layer of the showcase skill family.
